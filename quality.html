UPstart – Project Quality Evaluation

Key Metrics

User Interface Comfort (UX)
≥ 80% of students who tested the search engine rate the interface as comfortable and clear. This metric increases student engagement and makes the site easier to use.
User Activity (Repeated Use)
≥ 70% of students use the search engine several times per week. This metric shows the practical value of the project and confirms that students find it useful.

Test Scenarios

1. Keyword Search
Input / Steps: Enter "lecture schedule" in search and click "Search"
Expected Result: Current pages with lecture schedules appear
Reason: Verify basic search engine functionality
2. Search with Typo
Input / Steps: Enter "lectur schdule"
Expected Result: System suggests correction and shows relevant results
Reason: Verify user-friendliness for incorrect queries
3. Main Page Load
Input / Steps: Open main page on desktop and mobile devices
Expected Result: Page loads within 1.5 seconds
Reason: Verify loading speed and quality of user experience
4. External Links Verification
Input / Steps: Click on "Student Portal" link
Expected Result: Link opens the correct portal page
Reason: Verify correctness of external links
5. Feedback Form
Input / Steps: Submit a message through the "Feedback" form
Expected Result: Message successfully sent; user receives confirmation
Reason: Verify contact functionality and ability to report issues or suggestions
6. Cross-Browser Compatibility
Input / Steps: Open page in Chrome, Firefox, and Edge
Expected Result: Page displays correctly and all functions work
Reason: Verify interface universality for all users
